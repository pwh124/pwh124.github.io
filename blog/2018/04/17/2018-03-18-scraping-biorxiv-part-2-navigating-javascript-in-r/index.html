Scraping bioRxiv - Part 2: Navigating Javascript in R

By Paul Hook | April 17, 2018

tl;dr

The area on bioRxiv pages that displays where the paper was published is populated using Javascript when the page is loaded. The R package RSelenium allows for the scraping of anything created by Javascript, including the published paper’s DOI.

Where I left off

In my ongoing series of blog posts attempting to track the publishing of papers on bioRxiv,I left off with being able to scrape bioRxiv pages in R but not being able to extract the information about the published peer-reviewed version of the paper.

Why can’t I scrape the publishing information?



I had written about before that if a paper posted on bioRxiv goes on to be published in a peer-reviewed journal, that information (journal name, DOI number) appears on the paper’s bioRxiv page (seen above). However, when I went to scrape that information, it appears as “blank.” I can scrape other information off the page, why not that?

This question tested my very limited knownledge about websites and the HTML behind them. Luckily, I know how to use Google. I found this article great article from https://www.makeuseof.com/.

The short of it is that internet browsers use Javascript to load elements of webpages but only after the HTML has been parsed. The R package I am using (rvest) seems to only read/scrape the HTML code of a website, anything that is dynamically loaded by Javascript in a browser will not be accessed.

Using RSelenium with rvest to scrape bioRxiv

Fortunately, there are ways to mimic a webpage being loaded in a browser.

Again, through some creative Google-ing, I found that one of the ways to scrape Javascript loaded elements from websites in R is by using a combination of rvest and something called PhantomJS. Shout out to Florian Teschner at R-bloggers and Github user hrbrmstr for the help.

PhantomJS is what is called a “headless” browser which means that it has no traditional graphical user interface (GUI) and is mostly control through the command-line (Source: Wikipedia). Headless browsers are often used for scraping so PhantomJS sounded perfect for what I needed.

Unfortunately, the initial ways I found to use PhantomJS and rvest in R (links above), used a program/script that was somewhat external to R. Since I am stubborn, I wanted to use PhantomJS in the R environment. As with most things you want in R, there’s a package for that, RSelenium.

As I soon found, one can use RSelenium to headlessly browse a webpage using PhantomJS (vingnette here).

In order to use this method, PhantomJS needed to be installed in my path. This can be done easily with the package manager Homebrew and the following command:

Pauls-MacBook-Pro-2:~ paulhook$ brew install phantomjs

Now, I can start to headlessly browse.

library(RSelenium)
library(rvest)
## Loading required package: xml2
library(magrittr) # This package will help with some text manipulation I'll do in a second

The RSeleniium documentation was very helpful when trying to do this and my strategy is taken from there.

# https://cran.r-project.org/web/packages/RSelenium/vignettes/RSelenium-headless.html

# Connecting with the Selenium server
pJS <- phantom() # start the phantom webdriver
Sys.sleep(10) # the system needs to pause while the phantom binary loads
remDr <- remoteDriver(browserName = 'phantomjs') # communicate with the Selenium server
remDr$open(silent = T) # open the connection

# Use RSelenium to go to the webpage I want
remDr$navigate('https://www.biorxiv.org/content/early/2015/03/04/015859')

# Grab the title of the webpage
remDr$getTitle()[[1]]
## [1] "Modeling Linkage Disequilibrium Increases Accuracy of Polygenic Risk Scores | bioRxiv"

I have successfuly navigated to the bioRxiv page and I can see that the page has the correct title. Now I would like to get the entire page and read the HTML so I can scrape any component I want from the page. That can be done using the “remoteDriver$getPageSource()” and “read_html()” commands.

# Grab the documents underlying the website
hlink <- remDr$getPageSource()[[1]] %>%
  read_html()

Now that I have grabbed the HTML to scrape, the connection to the Selenium server and PhantomJS can be closed.

remDr$close()
pJS$stop()

I can now start scraping what I want with rvest. First, I can scrape the title of the paper.

#Title of the paper
title <- hlink %>%
  html_node("#page-title") %>%
  html_text()

title
## [1] "Modeling Linkage Disequilibrium Increases Accuracy of Polygenic Risk Scores"

And I can move on to trying to scrape the publication information on the page and format the paper’s peer-reviewed DOI number, which will provide a unique identifier for the published version of the paper.

pub_string <- hlink %>%
    html_nodes(".pub_jnl") %>%
    html_text() %>%
    extract(1) %>%
    strsplit(" ") %>%
    unlist()

pub_string
##  [1] "Now"                        "published"                 
##  [3] "in"                         "American"                  
##  [5] "Journal"                    "of"                        
##  [7] "Human"                      "Genetics"                  
##  [9] "doi:"                       "10.1016/j.ajhg.2015.09.001"
if("preprint" %in% pub_string){
  pub_doi <- "NA"
} else {
  pub_doi <- tail(unlist(strsplit(pub_string, " ")),1)
}

pub_doi
## [1] "10.1016/j.ajhg.2015.09.001"

By scraping the website after Javascript elements have been loaded, the bioRxiv page has now been successfully scraped for the publication information. With this ability, any preprint on bioRxiv can now be linked to the DOI for the peer-reviewed version of the paper. With the DOI in hand, all the information about this paper can now be parsed and linked to the preprint as well.

Automating this approach and pulling out all the information that would link a preprint to the peer-reviewed version of a paper will be the subject of my next blog posts.
